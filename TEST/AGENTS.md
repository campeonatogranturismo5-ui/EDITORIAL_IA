# Instrucciones para `TEST/`

- Las pruebas deben ser reproducibles y no depender de secretos.
- Separa pruebas unitarias, integración, casos documentales y pruebas sintéticas.
- Incluye casos válidos, inválidos, rechazo, reintento y escalado cuando corresponda.
- No modifiques datos reales ni servicios externos desde una prueba documental.
- Registra con precisión pruebas superadas, fallidas y no ejecutadas.
