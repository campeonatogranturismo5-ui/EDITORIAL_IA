# Instrucciones para `LIBROS/`

- Cada proyecto debe tener PID válido, estado explícito, log, decisiones y checkpoints.
- No mezcles archivos entre proyectos activos o archivados.
- No sobrescribas manuscritos ni paquetes: crea una nueva versión.
- No avances de estado sin artefactos obligatorios ni aprobación humana aplicable.
- No uses contenido real o publicable cuando la tarea autorice solo una simulación.
